Project Implementation: Retrieval Methods on the Cranfield Dataset
This implementation explores four main retrieval ideas on the Cranfield dataset: Query Reduction, Unordered Local-Context Bag-of-Words, Word Mover’s Distance (WMD) reranking, and SIF-based dense embeddings.
An important detail is that the Query Reduction module contains four experimental variants: idf_topk_legacy, idf_topk_extended, prf_term_pruning_legacy, and prf_term_pruning_extended. All methods are evaluated against a unigram TF-IDF baseline using cosine similarity on stopword-removed and normalized documents and queries.
________________


2. Query Reduction
Query reduction assumes that not every query term contributes equally to relevance. The goal is to prune less useful terms (generic, ambiguous, or weakly helpful) before retrieval so ranking focuses on the most discriminative parts of the query. There are two underlying strategies spanning four implemented versions.
2.1 IDF Top-k Reduction
This method keeps only the top "k" query terms with the highest IDF values, following the intuition that rare terms are more informative. For a query term "t", the score is:
* score_idf(t) = idf(t)
The top "k" highest-scoring terms are retained, and the reduced query is ranked using the baseline model.
* Best Parameter: k = 20 (Average reduced query length: 10.22)
Insight: Both versions effectively matched the baseline. On Cranfield, simple IDF-based pruning is not strong enough to improve retrieval because the queries are already short and technically precise.
2.2 Pseudo-Relevance-Feedback (PRF) Term Pruning
This method uses a first-pass retrieval step. Each original query term is rescored based on how strongly it is supported in the top-retrieved documents (D_m).
* support(t) = ( 1 / |D_m| ) * Σ [ tf(t,d) * idf(t) ]
This is combined with the original IDF score to find the final score, preserving the top "k" terms:
* final_score(t) = α * idf(t) + (1 - α) * support(t)
Insight: While the extended version was the best reduction variant, gains over the baseline were marginal (ΔMAP@10 = +0.0006). Pruning technical vocabulary often removes signal instead of noise.
________________


3. Unordered Local-Context Bag-of-Words
This method extends TF-IDF by adding local context features. It examines nearby-word windows (up to radius "n" left and right) around each token. Unique tokens inside the window generate unordered combinations (pairs/triples) which become binary context features.
* S(q,d) = S_base(q,d) + α * S_ctx(q,d)
________________


4. Word Mover’s Distance (WMD) Reranking
WMD is a semantic distance method based on GloVe 50d word embeddings. It measures the "movement" required to transform the query word distribution into the document word distribution.
* WMD(q,d) = min_T Σ [ T_ij * ||x_i - y_j||_2 ]
Because WMD is computationally expensive, the system retrieves a candidate set using TF-IDF and reranks those candidates by WMD.
________________


5. SIF Embeddings
Smooth Inverse Frequency (SIF) builds dense embeddings for documents and queries by averaging GloVe 50d word vectors with frequency-based downweighting (rare words contribute more).
* v_s = ( 1 / |s| ) * Σ [ (a / (a + p(w))) * e_w ]
To reduce common-direction noise, the top principal component is removed:
* v'_s = v_s - UU^T v_s
________________


6. Overall Comparative Interpretation
The experiments show that on Cranfield, the best improvements come from carefully enriching lexical retrieval with contextual structure rather than replacing it with dense semantic methods.
Overall Ranking of Methods:
1. Unordered Local-Context BoW: Best overall; clearly above baseline.
2. PRF Term Pruning (Extended): Best query-reduction variant; marginally above baseline.
3. Baseline TF-IDF / IDF Top-k: Essentially tied; forms the performance floor for effective lexical matching.
4. Pure WMD: Weaker than baseline; struggles with technical exactness.
5. Pure SIF: Weakest on this dataset; smooths away critical specialized vocabulary.
Conclusion: For technical information retrieval, hybrid lexical-context methods are vastly more effective than pure semantic methods. Simple query pruning is only beneficial when the original query contains genuine noise.

Regarding the pipeline for query expansion, we represent queries as term-frequency vectors rather than through IDF weighting. For every source term, candidate neighbors are gathered based on similarity scores. We then implement a two-stage thresholding process: first, a method-level floor is established using the maximum of a base minimum or a global quantile across all neighbor scores. Second, we filter neighbors for each query term using a threshold derived from the method floor, mean scores, and local quantiles. Following this, min-max scaling is applied to the remaining neighbors. These values are then converted into L1-normalized weights; notably, this avoids softmax since exponentials are not utilized. During the formation of the expanded query, the original mass remains constant while the expansion mass is distributed among neighbors. Consequently, the original word count does not diminish during expansion, as self-matches are excluded. Any shift in relative importance occurs only during retrieval when IDF multiplication and L2 normalization are applied for final cosine scoring.

LSA excels when:
1. Weak lexical overlap: Relevant documents use different terminology for same concept

LSA struggles when:
1. Sparse semantic signal from insufficient term co-occurrence
2. Completely disjoint concepts with no latent semantic connection
3. Query specificity exceeds 250-dimensional latent space capacity

Hybrid System Architecture
The best-performing configuration combines LSA with baseline TF-IDF:
hybrid_score(q,d)=0.2⋅tfidf_score(q,d)+0.8⋅lsa_score(q,d)
Rationale:
* TF-IDF component (20%): Captures exact term matching, proper nouns, specific phrases
* LSA component (80%): Captures semantic relationships and conceptual proximity
* Balance: TF-IDF acts as "anchor" preventing pure semantic drift while LSA provides semantic power
Performance:
* Hybrid essentially matches LSA while providing additional robustness
Known Limitations
LSA Limitations:
1. Static vocabulary: No out-of-vocabulary handling for new terms
2. Linear assumptions: SVD assumes linear topic-term relationships
3. Fixed dimensionality: 250 dimensions may insufficient for extremely fine-grained distinctions
4. No ranking diversity: Returns semantically similar but redundant documents
ESA Limitations:
1. Concept space sparsity: 92% zero entries limit ranking expressiveness
2. Concept adequacy: Performance bounded by document collection coverage
3. Explicit representation: Cannot discover abstract relationships beyond document similarities
4. Scalability: Full concept similarity computation doesn't scale to millions of documents
Part VI: Conclusion
We have implemented and comprehensively analyzed two complementary semantic retrieval paradigms for the Cranfield benchmark collection:
Key Insights:
1. Latent > Explicit: LSA's learned 250-dimensional topic space outperforms ESA's explicit 1398-document concept space, indicating that learned dimensionality reduction captures semantic structure more effectively than explicit document similarity.
2. Query Type Matters: LSA dominates on broad, multi-faceted queries; ESA occasionally excels on narrow, terminology-specific queries.
3. Feature Richness Trade-off: ESA's 24× larger feature set (81k vs 3.4k) provides occasional gains but adds computational cost without overall improvement.
4. Hybrid Robustness: Combining LSA's semantic power with TF-IDF's lexical precision achieves best-in-class performance while remaining robust to diverse query characteristics.
5. Domain Specificity: The aeronautics domain benefits significantly from semantic retrieval; 63% of queries improve with LSA, indicating strong latent semantic structure in technical literature.
This comprehensive analysis provides a foundation for:
* Understanding when latent vs. explicit semantic methods are appropriate
* Selecting optimal methods for specific retrieval scenarios
* Implementing production-grade semantic retrieval systems
* Guiding future research in semantic information retrieval
The detailed tuning methodology, systematic hyperparameter evaluation, and extensive query-level case studies enable practitioners to replicate and extend these results across other document collections and domains.

Sudheer Methods: Word2Vec-based Semantic Retrieval
Goal
* Improve baseline TF-IDF + cosine similarity
* Target limitation: exact keyword matching
* Try to handle semantically related words, synonyms, and wording mismatch between query and document
Methods used
1. word2vec_average
2. word2vec_idf_weighted
3. soft_cosine_word2vec
1. Word2Vec Average
* Train Word2Vec on Cranfield documents
* Represent each document and query by averaging all word embeddings
* Rank using cosine similarity between dense vectors
What it tries to solve
* Reduces dependence on exact word overlap
* Can retrieve semantically similar content even if wording changes
Limitations
* Entire query/document becomes one vector
* Important rare terms may get diluted
* No phrase order or local context
Pros
* Simple method
* Easy to explain
* Basic semantic retrieval baseline
Cons
* Weakest among the three methods
* Loses detailed term information
* Performs below TF-IDF baseline
2. Word2Vec IDF-Weighted
* Train Word2Vec on Cranfield documents
* Weight each word vector using TF-IDF importance before averaging
* Rank using cosine similarity
What it tries to solve
* Gives more importance to rare and meaningful technical terms
* Improves over plain averaging
Limitations
* Still compresses everything into one vector
* Still loses phrase structure and token order
Pros
* Better than plain Word2Vec Average
* Preserves important terms better
* Stronger dense retrieval method
Cons
* Still below TF-IDF baseline
* Semantic gain is limited by single-vector representation
* Allows semantic term matching
* Best balance of lexical and semantic retrieval
3. Soft Cosine with Word2Vec
What this does
* Keeps the normal TF-IDF representation for queries and documents
* Uses Word2Vec to measure similarity between terms
* Uses soft cosine so related words can partially match, not only exact same words
Formula
* Baseline cosine: cos(q,d) = (q.d) / (|q||d|)
* Soft cosine: soft(q,d) = (q^T S d) / sqrt((q^T S q)(d^T S d))
* Here S is the term similarity matrix built using Word2Vec
What this tries to solve
* Reduces exact keyword matching problem in TF-IDF
* Helps when query and document use different but related words
* Adds semantic matching without removing TF-IDF structure
Limitations addressed
* Lack of semantic understanding in plain TF-IDF
* Poor handling of synonyms or related technical terms
* Failure of exact-match retrieval when wording changes
Why it is the best among these three
* word2vec_average and word2vec_idf_weighted convert the whole text into one dense vector
* That can lose important term-level details
* Soft cosine keeps term-level TF-IDF information and adds semantic similarity
* So it gives the best balance between lexical and semantic retrieval
Pros
* Best method among Sudheer’s three methods
* Keeps TF-IDF interpretability
* Allows partial matching of related words
* Better balance of lexical and semantic retrieval
Cons
* Higher runtime
* Needs tuning of neighbor count and similarity threshold
* Too many neighbors can add noisy matches
* Still slightly below baseline TF-IDF overall
Short conclusion
* Best semantic method in Sudheer’s folder
* Most practical hybrid method among the three
* Good improvement over strict exact-match retrieval
